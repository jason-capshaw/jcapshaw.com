## Design Context

### Users
Practitioners and decision-makers in B2B distribution, enterprise commerce, and applied AI. They arrive from search or social links, looking for substantive thinking on real-world technical problems — not tutorials, not thought-leadership fluff. They're reading on laptops during work hours or phones during commutes. The job: find a credible perspective that helps them think through their own challenges.

### Brand Personality
**Grounded. Sharp. Restrained.**

The voice is a senior practitioner writing for peers — direct, opinionated, no hedging. The interface should feel like opening a well-typeset essay, not visiting a website. Every design decision serves readability and credibility. Nothing decorative, nothing performative.

### Aesthetic Direction
- **Visual tone:** Editorial minimalism. Warm, not cold. Serious, not sterile.
- **Palette:** Monochromatic warm stone neutrals. No accent color. Color comes from the writing, not the chrome.
- **Typography-driven identity:** Geist Sans/Mono. No logo — the name in type IS the brand.
- **References:** Stripe Press editorial pages, Craig Mod's writing, iA Writer's reading experience.
- **Anti-references:** Medium (too much chrome), typical dev blogs (code-heavy aesthetics), corporate marketing sites (stock photos, CTAs everywhere), anything with gradient hero sections or floating UI elements.
- **Theme:** Light and dark mode via `prefers-color-scheme`. Both are first-class.

### Design Principles

1. **Content is the product.** Every pixel exists to make the writing easier to read. If a design element doesn't serve comprehension or navigation, remove it.

2. **Earn trust through restraint.** No animations, no decorative elements, no unnecessary interactions. The confidence is in what's absent. Substance over spectacle.

3. **Warm, not cold.** The palette is stone and earth, not blue-gray tech. The spacing is generous, not tight. The type is large, not compact. The site should feel like a well-made book, not a dashboard.

4. **Ship fast, stay minimal.** Vanilla CSS with design tokens. No component libraries, no CSS frameworks. The smallest amount of code that achieves the design intent. Performance is a feature.

5. **Accessible by default.** WCAG AA minimum. High contrast, semantic HTML, responsive typography, full dark mode support. No motion dependencies.

### Component Patterns
- **Header:** Name + monospace tagline left, nav right. Horizontal rule below. Mobile hamburger at 768px.
- **Footer:** Two-column — description left, connect links right. Stacked on mobile.
- **Article list:** Two-column grid — title/excerpt left, type/date right. Single column on mobile.
- **Article:** Type label, title (2.25rem), date + reading time meta. Body at 1.1rem/1.8 line-height. First paragraph slightly enlarged.
- **Page intro:** Headline (2rem) + optional subhead, bottom border.

### Responsive Strategy
- Single breakpoint at 768px (tablet/mobile)
- Secondary at 480px for tighter mobile adjustments
- Base font: 18px desktop, 17px tablet
- All layouts shift from multi-column to single-column at 768px
